# Mobile Prototype Notes

## Prototype Asset

Low-fidelity mobile wireframes are provided here:

- `mobile-wireframes.svg`

## Screen Set

The prototype focuses on the six warehouse-critical screens:

1. Home
2. Search Material
3. Search Pallet
4. Inbound
5. Outbound
6. Cycle Count and Logs

## Interface Rules Reflected In The Prototype

- Search Material and Search Pallet are the highest-priority entry points.
- All key actions use large thumb-friendly buttons.
- Inbound is compressed into a three-step vertical form.
- Outbound shows FIFO recommendation before confirmation.
- Search results prioritize current stock visibility over administrative metadata.
- Clear pallet is visually separated as a dangerous action.

## Visual Direction For Coding

- Bright, industrial, high-contrast palette
- Large numeric and code typography
- Dense information only after search results, not on the home screen
- Avoid desktop-style tables as the first interaction pattern on mobile

## Review Focus

When you review the wireframe, the main confirmation points should be:

- Is the home screen hierarchy right for warehouse staff?
- Are material and pallet lookup prominent enough?
- Are inbound and outbound really short enough?
- Is the result layout clear enough under fast-paced usage?
